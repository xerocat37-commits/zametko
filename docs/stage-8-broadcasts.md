## Архитектурные решения

- **Список Global берётся из `/api/notes/feed` с локальной фильтрацией.** На бэкенде отдельного `GET /api/global` нет, потому что:
  - Для Student / Teacher / Head — Global уже включены в `/feed`.
  - Дублировать эндпоинт ради Head — лишний код.
  - Фильтр `NoteType=Global` на клиенте — одна строка LINQ.
- **Очистка формы после успешного POST** через `NewTitle = string.Empty; NewContent = string.Empty;`. Не использовал отдельную команду «Очистить» — после успеха пользователь хочет писать следующую рассылку, форма должна быть готова.
- **Заглушка → реальная страница без изменения типа `HeadBroadcastPage`.** AppShell.xaml на этапе 6 уже ссылался на этот класс; смена только содержимого XAML и code-behind, маршрут и видимость по роли работают без правок.

### Сквозной сценарий ТЗ (end-to-end)
1. Чистая БД → запуск API → Head создаётся seed-ом.
2. Логинимся как Head (`Заведующий` / `head-admin`) → во Flyout видна «Рассылка».
3. Создаём рассылку «Завтра выходной» → появляется в истории.
4. Logout → регистрируемся как препод «Иванов И.И.».
5. Создаём группу «ИВТ-21», приглашаем «Петров П.П.», создаём групповую заметку «Лекция 5».
6. Logout → регистрируемся как студент Петров П.П. с группой ИВТ-21.
7. Открываем «Ленту» → видим обе записи: и «Завтра выходной» (Global), и «Лекция 5» (Group).
8. Создаём личную заметку → видна только нам.
